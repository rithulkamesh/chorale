# v1.2 — Modular signal graph ("the canvas")

> **Shipped (v1.2.0) — how reality diverged from this design.** The graph
> landed as a fixed 41-id node pool (8 voices, 8×EQ/COMP/SAT, 4 GAIN,
> 2 ECHO, 2 REVERB, OUT) with portless edges — every input is a summing
> junction, so there are no Sum/Split nodes and no dynamic allocation.
> Echo and reverb shipped as wireable nodes and the per-voice send knobs
> (and the wet-bus echo/reverb + master REVERB module) were removed;
> pre-1.2 states migrate to an equivalent wired patch on load. LeadIn /
> Pan / StemOut nodes were dropped — stems and scopes tap the end of each
> voice's lane instead. The default patch is voices → OUT (FX modules are
> bypassed by default, so this is audibly identical to the old chains);
> "wire classic chains" is a right-click action. The rest of this document
> is the original design, kept for the reasoning.

Goal: replace the fixed per-voice chain with a patchable graph. Users get a
canvas of blocks — voice inputs, effects, summers, sends, outputs — and wire
them however they want. The current chains become nothing more than the
default patch.

## Node model

```
Node
├─ id            stable int, allocated once, never reused
├─ type          VoiceIn | LeadIn | Eq | Comp | Sat | Echo | Reverb
│                Gain | Pan | Sum | Split | MainOut | StemOut
├─ params        owned instances, namespaced "n<id>.<param>"
└─ position      canvas x/y (UI only)

Edge: {fromNode, fromPort, toNode, toPort}   // audio, mono or stereo port
```

- **Sum** is just a node with N inputs summed to one output; **Split** mirrors
  one input to N outputs. These two + Gain/Pan give the "wire and experiment"
  freedom without special cases.
- Graph is a **DAG** — cycles rejected at wire time (visual feedback on the
  cable). Feedback loops are the one deliberate exclusion for v1.2: they need
  a one-block delay contract and explode the testing surface.
  (`ponytail:` revisit if users ask for dub-style echo feedback patching.)

## Engine

- Topological sort once per graph edit (message thread), swap into the audio
  thread as an immutable "compiled plan" (vector of {nodeRef, input buffer
  indices, output buffer index}) via atomic pointer swap — same pattern the
  settings snapshot uses today.
- Fixed block processing over the plan; one float buffer pool, indices
  assigned at compile time (no allocation on the audio thread).
- All DSP nodes reuse the existing JUCE-free modules (`ChannelEq`,
  `Compressor`, `Saturator`, echo ring, `SimpleReverb`) — the graph is a
  routing layer, not new DSP.
- Latency: all sources (voices) already share PSOLA latency; pure-FX nodes
  are zero-latency, so no per-path compensation needed in v1.2. If a
  latency-adding node ever lands, the compile step gains a delay-balancing
  pass.

## Parameters & state

- Dynamic node count vs. APVTS's fixed layout: pre-allocate a **node pool**
  (e.g. 32 nodes × worst-case param set) at construction, park unused nodes'
  params at defaults and hide them from the host list where possible. This
  keeps automation working in every host without dynamic-parameter tricks.
- Graph topology (nodes, edges, positions) serializes as a child ValueTree of
  the existing state — rides along with presets, A/B, and undo for free.
- Migration: on load of a pre-1.2 state, synthesize the default patch
  (voices → chains → main out) and map old param values onto pool nodes.

## Canvas UI

- One `GraphCanvas` component in the FX view (replacing the two fixed
  ChainStrips): pan/zoom (transform, same trick as the resizable editor),
  nodes as the existing module cards, bezier cables between ports.
- Drag from port to port to wire; drag a cable off a port to disconnect;
  right-click canvas → add-node menu; double-click node → its existing editor
  (EqPanel / CompPanel / SatPanel / …) opens in a side panel. All module
  editors already exist and bind by param prefix, so they plug straight in.
- Keep the current Stage/Mixer views untouched — the canvas is power-user
  territory; the default patch means nobody *has* to see it.

## Slices (each shippable)

1. Engine: compiled-plan executor behind the current fixed chains (no UI
   change, proves the swap machinery).
2. Node pool params + serialization + migration.
3. Canvas UI read-only (view the default patch).
4. Wiring + add/remove nodes + Sum/Split/Gain/Pan.
5. Polish: cable animations, per-cable level glow, patch presets.

Estimate: the engine slices (1–2) are a week-ish of careful work; the canvas
(3–5) another two. Nothing here blocks 1.1.x maintenance.
